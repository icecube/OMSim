In the folder {\ttfamily simulations/wavepid} you find the files pertinent to the Wave\+PID photon origin tracking study.\hypertarget{md_extra__doc_233__wavepid_autotoc_md64}{}\doxysection{\texorpdfstring{Introduction}{Introduction}}\label{md_extra__doc_233__wavepid_autotoc_md64}
The Wave\+PID simulation studies photon arrival time distributions at Ice\+Cube optical modules. A primary particle (typically a muon) is injected as a straight track passing the \doxylink{class_d_o_m}{DOM} at a configurable perpendicular distance (impact parameter) and energy. Along its path, the particle produces Cherenkov radiation and electromagnetic secondaries (e.\+g. brems-\/photons or pair-\/produced positrons and electrons). Each optical photon that reaches the \doxylink{class_d_o_m}{DOM} photocathode is recorded together with its origin — the production mechanism and parent particle type — enabling classification into categories such as "{}\+Cherenkov from Muon"{}, "{}\+Cherenkov from Electron"{}, "{}\+Scintillation"{}, etc. These photon arrival time distributions form the basis of the Wave\+PID particle identification method.

\begin{quote}
{\bfseries{Note\+:}} This study was applied and tested solely with {\ttfamily -\/-\/detector\+\_\+type 3} (standard \doxylink{class_d_o_m}{DOM} with normal quantum efficiency). Other module types are technically supported via the {\ttfamily -\/-\/detector\+\_\+type} flag but have not been used or validated in the scope of the Wave\+PID study. The {\ttfamily -\/-\/efficiency\+\_\+cut} flag was not used — all photons arriving at the sensitive volume were recorded, as the QE-\/based die roll affects all photon origins equally and does not impact the origin-\/based timing distributions. \end{quote}

\hypertarget{md_extra__doc_233__wavepid_autotoc_md69}{}\doxysubsection{\texorpdfstring{Command-\/\+Line vs. Macro File Parameters}{Command-Line vs. Macro File Parameters}}\label{md_extra__doc_233__wavepid_autotoc_md69}
The particle source can be configured in two ways\+:


\begin{DoxyEnumerate}
\item {\bfseries{Command-\/line arguments}} ({\ttfamily -\/d}, {\ttfamily -\/e}, {\ttfamily -\/p})\+: The {\ttfamily \doxylink{class_o_m_sim_primary_generator_action}{OMSim\+Primary\+Generator\+Action}} uses these to set up a GPS particle source with the correct geometry (impact parameter, Cherenkov cone alignment).
\item {\bfseries{Macro file}} ({\ttfamily -\/m})\+: Provides full control over the GPS configuration via Geant4 macro commands. When a macro file is provided, its GPS commands override the command-\/line particle settings.
\end{DoxyEnumerate}

For simple configurations, command-\/line arguments are sufficient. For complex setups (multiple particle sources, custom angular distributions), use a macro file.\hypertarget{md_extra__doc_233__wavepid_autotoc_md70}{}\doxysection{\texorpdfstring{Usage Examples}{Usage Examples}}\label{md_extra__doc_233__wavepid_autotoc_md70}

\hypertarget{md_extra__doc_233__wavepid_autotoc_md77}{}\doxysection{\texorpdfstring{Notes on World Volume Size}{Notes on World Volume Size}}\label{md_extra__doc_233__wavepid_autotoc_md77}
The world volume is a sphere of 30 m radius. For simplicity, no attempt was made to tune the world size precisely — the chosen radius fully contains the hadronic/electromagnetic cascade and a sufficiently large segment of the muon track for the target energy range of 1–100 GeV, without a noticeable performance impact.\hypertarget{md_extra__doc_233__wavepid_autotoc_md78}{}\doxysection{\texorpdfstring{Notes on $<$tt$>$-\/-\/efficiency\+\_\+cut$<$/tt$>$}{Notes on <tt>--efficiency\_cut</tt>}}\label{md_extra__doc_233__wavepid_autotoc_md78}
The Wave\+PID study did {\bfseries{not}} use {\ttfamily -\/-\/efficiency\+\_\+cut}. All photons arriving at the sensitive volume (PMT photocathode) were recorded regardless of detection probability. This is because the QE-\/based die roll affects all photon origins equally and therefore does not impact the origin-\/based timing distributions that are the focus of this study.

The flag can be enabled if desired\+:


\begin{DoxyItemize}
\item {\bfseries{Without}} {\ttfamily -\/-\/efficiency\+\_\+cut} (default, used in Wave\+PID study)\+: All photons reaching the photocathode are recorded. The detection probability is stored in the output and can be applied as a weight in post-\/processing.
\item {\bfseries{With}} {\ttfamily -\/-\/efficiency\+\_\+cut}\+: Only "{}detected"{} photons are recorded (detection probability set to 1 in output). This reduces output file size but prevents re-\/weighting in analysis.
\end{DoxyItemize}\hypertarget{md_extra__doc_233__wavepid_autotoc_md79}{}\doxysection{\texorpdfstring{Notes on Multithreading}{Notes on Multithreading}}\label{md_extra__doc_233__wavepid_autotoc_md79}
By default the simulation uses a single-\/threaded {\ttfamily G4\+Run\+Manager}. This is intentional\+: the Geant4 Qt GUI ({\ttfamily -\/v}) crashes in {\ttfamily G4\+MTRun\+Manager} mode because {\ttfamily G4\+Worker\+Run\+Manager\+Kernel\+::\+Setup\+Shadow\+Process()} fails for particles (such as alpha) that lack a process manager in worker threads during interactive sessions.

To enable multithreading for large batch runs (where the GUI is not needed), pass {\ttfamily -\/-\/multithreading}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{./OMSim\_WavePID\_study\ -\/n\ 1000\ -\/-\/detector\_type\ 3\ -\/-\/environment\ 2\ -\/d\ 5\ -\/e\ 30\ -\/-\/multithreading\ -\/-\/threads\ 8\ -\/o\ output}

\end{DoxyCode}


Do {\bfseries{not}} combine {\ttfamily -\/-\/multithreading} with {\ttfamily -\/v} — the GUI will crash.

When multithreading is active, each worker thread maintains its own hit data storage (using {\ttfamily G4\+Thread\+Local}), which is merged at the end of each run. The {\ttfamily Tracking\+Action} track-\/to-\/particle maps are also {\ttfamily thread\+\_\+local}, so no mutex is needed — each worker thread owns its maps independently.

Note that running with different thread counts may produce different event-\/to-\/thread assignments, so exact hit-\/by-\/hit reproducibility across different thread counts is not guaranteed. However, statistical results should be consistent.\hypertarget{md_extra__doc_233__wavepid_autotoc_md80}{}\doxysection{\texorpdfstring{Architecture}{Architecture}}\label{md_extra__doc_233__wavepid_autotoc_md80}
